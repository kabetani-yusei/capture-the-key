# Capture The Key

## 文章を要約するアプリ
文章を入力して「要約する」ボタンを押すと、
API にリクエストを送り、要約した文章を返すアプリです。

## 日記

### 2026/04/22
最近話題の Claude Code で、文章を要約するアプリを作ってみる。  
作ってほしいアプリの概要を入力したら、Claude Code が全部作ってくれた。すごい！  
最初はエラーが起きて、やっぱり一発では無理かと思っていたけど、  
[src/app/page.tsx](src/app/page.tsx) の `KEY2 = 'ここにあなたのKEY2を入れる'` をそのまま実行していたのが原因だった。  
自分の KEY2 を入れたらちゃんと動いて、動くものが作れて感動した！

### 2026/04/23
どうやらシークレットキーなどの外部に公開されたら困るものを、直接コードに書き込むのは危険らしい。  
そこで、これまで使っていた KEY2 は使えないようにして、新しく KEY3 を発行することにした。  
[.env](.env) ファイルを新しく作り、キーはそちらに書いて、[src/app/page.tsx](src/app/page.tsx) からは `process.env` 経由で読み込むようにしてみた。  
これでコードに直接キーを書かなくて済むので安心！

### 2026/04/24
どうやら [.env](.env) ファイルも、そのまま公開したらダメらしい。  
そこで、これまで使っていた KEY3 は使えないようにして、新しく KEY4 を発行して [.env](.env) に書き込んだ。  
今度こそ公開しないように注意しながら他の変更点を push した。  
あっ、間違えて [.env](.env) も push されちゃった！  
まあ KEY4 を削除して push し直せば大丈夫か。  
一応ファイルを確認してみたら、うん大丈夫、KEY4 は見えていない。これで安心！

### 2026/04/25
どうやら [.env](.env) から KEY4 を削除して push し直しても、履歴を辿れば過去のコミットから見えてしまうらしい。  
そこで、これまで使っていた KEY4 は使えないようにして、新しく KEY5 を発行することにした。  
今度は同じ失敗を繰り返さないように、そもそも [.env](.env) を push しないようにしてっと。  
あとは、せっかくだしいろんな人に使ってもらいたいからデプロイもしたいな。  
こういうときには Vercel などを使えばよいのか。  
Vercel でデプロイするときは管理画面の Environment Variables に KEY5 を登録すれば、コードからは `process.env` 経由でそのまま参照できるらしい。  
これでコードにもリポジトリにもキーが残らないので、今度こそどこからも見られないはず。完璧！
